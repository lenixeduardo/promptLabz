# To Issues

Decomposes plans, specs, or PRDs into actionable GitHub issues:
- Uses vertical slicing for independent, shippable units
- Adds labels, milestones, and dependencies
- Ensures each issue has clear acceptance criteria
- Maintains traceability back to the original spec
