## Riscos e Observações

1. **`.gitignore` corrompido (CRÍTICO):** O arquivo de regras de exclusão do Git
   foi destruído. Recomenda-se verificar imediatamente o estado atual na `main` e
   restaurar as regras originais se ainda estiverem ausentes.

2. **Descrição de PR insuficiente:** A PR foi criada sem descrição técnica, apenas
   com referência interna ao task ID. Isso impede rastreabilidade futura e dificulta
   revisão de código significativa.

3. **Merge sem revisão de diff:** A corrupção do `.gitignore` passou despercebida,
   indicando que o diff não foi revisado antes do merge. PRs geradas por agentes
   automáticos requerem revisão humana obrigatória do diff completo.

4. **Agente externo sem guardrails:** O qwen-chat coder não validou o output antes
   de criar a PR. Um agente RAG Monitor integrado ao pipeline poderia ter detectado
   e bloqueado este tipo de corrupção.

## Lições Aprendidas

- **Chain-of-thought bleeding** em LLMs pode corromper arquivos de configuração
  críticos quando o modelo é instruído a gerar ou modificar arquivos baseando-se
  em contexto de diff.
- PRs automáticas de agentes externos devem ter **validação obrigatória de sintaxe**
  antes do merge (ex: `.gitignore` não pode conter linhas entre parênteses).
- O agente RAG Monitor deve ser integrado como **check de PR** para detectar
  regressões estruturais em arquivos de configuração.
- A descrição da PR deve incluir, no mínimo: o problema resolvido, os arquivos
  tocados e o racional de cada mudança.
